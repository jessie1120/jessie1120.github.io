專案實作功能說明

這次實作的專案主要功能是利用語音辨識，與網頁進行互動玩一個猜數字遊戲
目前支援語音辨識輸入的瀏覽器為Google Chrome

實際需要實做的功能：

  • 隨機產生數字作為遊戲目標(答案)
  • 取得語音輸入辨識的結果，並放置到畫面中
  • 需要提示目前的數字與目標的差距大小
  • 答對後顯示答案並產生可再玩一次的按鈕

Javascript Part 1 - generateRandomNumber 隨機數字函式

這邊用到的一樣是之前產生亂數的方式，並把產生的結果丟到變數randomNumber裡
運算的最後＋1是因為產生的數字會是0 - 99 而我們需要1 - 100 所以加一。

Javascript Part 2 - Get words form voice input 取得語音輸入

這個部分我們要取得語音辨識的結果result並新增到畫面上

首先我們先確定可支援的瀏覽器，並將新增語音辨識功能放置到變數recognition中

接著透過.start()方法啟動語音辨識 => 跳出通知需使用者同意取用麥克風

建立事件監聽取得語音辨識結果result

這邊稍微利用console.log看一下我們取得了什麼東西
下圖是成功啟用語音辨識後會取得的：
紅框部分是result的整體內容
而我們需要的部分是
第一層：SpeechRecognitionResultList內的
第二層：SpeechRecognitionResult內的
所在陣列：SpeechRecognitionResultAlternative中的transcript這個元素

所以我們這邊取得元素寫法為e.results[0][0].transcript;
第一層：[0] (第一個陣列索引值為0)
第二層：[0] (第一個陣列索引值為0)
第三層：.transcript (陣列中的transcript元素)

接著利用showMessage函式將我們取得的元素加入畫面(DOM)中

Javascript Part 3 - checkNumber 判斷數字函式

接著除了將我們剛剛語音的元素放到畫面上還要進行判斷來提示跟目標的差距
這個函式的實行步驟如下：

  1. 利用+號確保語音輸入後的數字為數字型別
  2. 利用isNaN()來判斷語音輸入不是數字的情況，並提示取得的元素不是數字
  3. 判斷式 => 超出範圍數字 => 提示數字需要1 - 100之間
  4. if迴圈判斷大小或答對，並新增 提示 或 正確答案 至畫面上

    Javascript Part 4 - Continue Game Event 繼續遊戲事件

    最後新增兩個事件監聽：

第一個是讓我們判斷出結果後，可以重新開始新一輪的語音辨識，
沒有這個事件的話只能猜一次遊戲就會停止

接下來是猜對後我們在畫面上新增的reload按鈕要實際有reload的功能

這邊的事件監聽對象因為要等到猜對後才新增出來，所以我們先綁定在父元素上，
後續實際點擊時透過ID來確認我們點擊的位置 ＝ reload按鈕才作reload動作
