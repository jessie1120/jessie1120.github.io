Javascript Part 1 - addTransactionDOM & init function

這次一樣會比較原生寫法跟jquery寫法的小差異，
其實我自己的感覺主要的差別還是在變數宣告、方法使用兩部分。

原生的部分依然是從變數宣告開始

雖然jquery利用選擇器很方便，但為了方便，先把會遇到的選擇器放在上方，需要時直接複製就好。

addTransactionDOM() 列表添加函式

這個函式的步驟大致是這樣：

  1. 宣告變數+判斷式 => 加上正負號
  2. createElement() => 創建新的列表元素
  3. 宣告變數+判斷式 => 加上class正負屬性
  4. 列表內增加文字及按鈕(動態方式) => 文字、正負符號、金額數字皆為動態

– 這邊有一個Math.abs() => 傳回絕對值 => 這邊用意是去掉負數的減號才不會重複
5. appendChild() => 將剛剛創建的元素丟到指定的父元素裡。

jquery的部分主要是創建列表元素跟最後丟到指定父元素的方法不同。

  • .appendTo() = 將內容添加至某個選取器
  • .append() = 在某個選取器裡添加內容

append()跟appendTo()使用說明

init() 初始化函式

這邊會先用自行輸入的示範物件來顯示在畫面上，之後再加入動態創建的物件。
先將陣列丟進變數中，用forEach()將物件內資料先跑過一次，
最後呼叫函式讓頁面載入時可以直接顯示。
這個部分jquery幾乎是一樣的，就不重複說明。

Javascript Part 2 - updateValues function

這個函式的步驟大致是這樣：

  1. 處理陣列中金額amount的值 => 用map()方法處理：運算後回傳新陣列
    map()方法說明
  2. 計算累計金額 => 用reduce()方法處理：累加值並回傳
    reduce()方法說明
  3. 計算收入 => 用filter()方法處理：經過內部函式處理後，將通過之元素回傳為新陣列。
    filter()方法說明
  4. 計算支出 => 用filter()方法處理，最後乘上-1變負數。
  5. 將結果丟入內文，顯示在畫面上。
  6. 最後在把這個函式丟進init function，讓它初始就能夠更新資訊。

這邊影片的函式呢是利用箭頭函式寫法，
但我為了記錄邏輯讓自己不要搞錯，所以用比較原始的函式寫法，但輸出結果是沒變的。

我這邊的寫法看起來比較複雜，但比較符合我的原始邏輯思考方式。

Javascript Part 3 - Submit Event 提交事件

這個事件監聽，只是要在提交的時候跑一個函式addTransaction()

addTransaction 列表添加函式 & generateID 亂數產生ID函式

這兩個函式主要功能就是能夠將輸入的值轉為我們在畫面上呈現的狀態。

函式的步驟大致是這樣：

  1. 先判斷文字text或金額amout是否為空 => 若為空 =>顯示alert
  2. 若不為空 => 創建transaction陣列 =>包含id、輸入文字的值、輸入金額的值。
  3. 用push()方法處理陣列：每次輸入就持續增加陣列內的元素

-push()方法說明
4. 添加列表 => addTransactionDOM函式
5. 更新畫面金額變動 => updateValues函式
6. 最後的部分是讓預設變回空白。

這邊有個小地方要注意金額的部分amount.value使用+號將字串轉為數字型別

再來是ID的建立，這邊是一個創造亂數的函式generateID
原理是這樣的：

  1. Math.random()：會隨機產生出0~1之間的小數
  2. Math.floor()：會將小數點捨去並回傳小於等於所給數字的最大整數。
  3. 將亂數產生的小數乘上你需要的倍數
  4. 倍數 = 隨機整數的數量(乘100倍就會產生100個隨機的整數)

這邊我自己的寫法有用了一個縮寫1e5，這個意思是1後面加幾個0
所以1e5= 100000，可以避免太多0打錯。

亂數產生原理-參考文章傳送門

jquery的部分注意選擇器不要弄錯，還有取值的方法不同。

removeTransaction 刪除列表函式

這個函式的功能就是讓我們刪除列表，這個函式會綁定在 點擊刪除按鈕 的事件。
這邊的事件寫法採用inline方式，在按鈕的屬性內建立事件。

函式的步驟：

  1. 使用filter()方法處理：經過內部函式處理後，將通過之元素回傳為新陣列。
    => 在這個函式中會是這樣的，將相同ID的元素移除 (我們點擊的列表)
    => 只回傳不相同的元素 (我們沒有點擊的列表)
  2. 接著初始化init讓畫面更新。 這邊我一樣用我自己的思考邏輯寫函式才不會搞混，但箭頭函式真的很簡潔。

    Javascript Part 4 - Local Storage

    這邊做的功能是，在重整後可以保留之前的列表，
    將這些內容存在Local Storage本地儲存空間，在需要時可以再拿出來進行運用。
    同時也避免關閉分頁後的資料損失。

首先我們先建立一個變數localStorageTransactions將存在local storage的資料拉出來。
JSON.parse()：轉換拉出來的字串資料 => 陣列array

接著在建立一個變數transactions =>
判斷不為空 => 使用localStorageTransactions將資料轉為陣列
為空 => 建立空陣列[]

接著做一個函式updateLocalStorage()將輸入的資料陣列轉為字串存入Local Storage
使用JSON.stringify

最後在添加列表及刪除列表中加入這個函式，讓添加或刪除時可以更新Local Storage的狀態。
