CSS

並且做了一點點的RWD變化，讓螢幕尺寸較小時，可以跟著變更顯示畫面

Javascript Part

這次影片課程採用的是原生JS的寫法，而專案製作有改用jquery，這邊會比較一下寫法的不同。
這個專案主要是串接匯率API到網頁，再經過函式來計算匯率轉換的結果。

首先在原生JS的部分，一樣是需要先宣告我們所需要的變數，
但jquery在這個部分就不用做了，它的選擇器可以很快速地選到我們需要的DOM位置。

Calculate() 計算函式

這個函式的功能如下↓

  1. 先將節點的值丟到變數中以利後續在畫面顯示可以用動態的方式呈現。
  2. 使用fetch()這個方法先從外部API取得一個包含response的promise物件
  3. 使用.then()方法來處理這個物件，第一個.then()接收物件回傳給下一個.then()，
  4. 接著將收到的資料丟進變數，接著修改innerText在畫面上做出匯率呈現，
    這邊使用${}都是要讓它動態擷取我們目前選擇的幣值。
  5. 最後在做一個簡單的乘法算出金額，後面的toFixed()是讓它顯示到小數點後幾位的方法。 以jquery的寫法我們不用先宣告變數，直接用jquery取值的方式.val()來代替。
    修改內文的部分也是用.text()直接替代。
    由於上面的箭頭函式雖然簡潔但我會有點混亂所以還是用原來的function寫法。

    Change Event

    這邊是當我們在畫面上操作，有選擇不同的幣值或金額時可以即時修正，
    下方有補上一個函式呼叫讓我們畫面載入時就可以直接顯示當前的預設匯率。

jquery這邊的話比起上面先把變數都宣告好的寫法，稍微複雜一點，
但看起來比較直覺，不用去想現在這個變數代表DOM哪裡的節點位置。
事件寫法大致就是$('要選擇的節點位置').change(呼叫函式)

Swap Event

這個事件的功能是我們畫面上的Swap按鈕在點擊時能夠自動交換匯率1跟匯率2，步驟如下↓

  1. 先將匯率1的值 => 丟在temp變數
  2. 將匯率2的值改為匯率1的值
  3. 再將原來temp變數中儲存的匯率1的值改為匯率2的值
  4. 最後在呼叫函式進行重新計算

jquery的部分其實寫起來一模一樣，只是將前面原生.value這個方式全部改成jquery的.val()
前面的=進行賦值，這邊就改成直接丟進.val()的括號裡。
